# 옵시디언 AI 비서 v3.0 — 빌드 방법

이 문서는 옵시디언 AI 비서 v3.0을 직접 빌드하는 방법을 설명합니다.  
**비개발자라면 A 방법(GitHub Actions)을 따르세요.** 직접 코드를 빌드할 필요 없이 GitHub가 자동으로 해줍니다.

---

## A) GitHub Actions 빌드 (추천 — 비개발자도 가능)

내 컴퓨터에 Rust, Node.js 등을 설치할 필요 없습니다. GitHub 서버가 대신 빌드해 줍니다.

### 준비물
- GitHub 계정 (무료): https://github.com
- 이 프로젝트 폴더 전체

---

### 1단계: GitHub 계정 만들기 (이미 있으면 건너뛰기)

1. https://github.com 접속
2. **"Sign up"** 버튼 클릭
3. 이메일, 비밀번호, 사용자명 입력
4. 이메일 인증 완료

---

### 2단계: 새 저장소(Repository) 만들기

1. GitHub에 로그인한 뒤 오른쪽 상단 **"+"** 버튼 → **"New repository"** 클릭
2. **Repository name**: `obsidian-ai-assistant` 입력
3. **Visibility**: Public 또는 Private 둘 다 가능합니다. Private 선택 시 코드가 공개되지 않습니다.
4. **"Create repository"** 버튼 클릭
5. 저장소가 생성되면 `https://github.com/<내-아이디>/obsidian-ai-assistant` 형태의 주소가 됩니다.

---

### 3단계: Git 설치 (처음이라면)

**Windows:**
1. https://git-scm.com/download/win 에서 설치 파일 다운로드
2. 더블클릭 → 기본값 그대로 설치
3. 설치 후 시작 메뉴에서 **"Git Bash"** 실행

**macOS:**
- 터미널을 열고 `git --version` 입력
- 설치 안 되어 있으면 Xcode Command Line Tools 설치 창이 뜹니다. **"설치"** 클릭

---

### 4단계: 프로젝트 폴더를 GitHub에 올리기

터미널(Windows는 Git Bash)에서 아래 명령을 순서대로 실행합니다.

```bash
# 프로젝트 폴더로 이동 (실제 경로에 맞게 수정)
cd "옵시디언 개인 ai"

# Git 저장소 초기화
git init

# 모든 파일 스테이징 (API 키 파일은 .gitignore로 자동 제외됨)
git add .

# 첫 번째 커밋 만들기
git commit -m "Initial commit"

# 기본 브랜치 이름을 main으로 설정
git branch -M main

# GitHub 저장소와 연결 (<your-username> 부분을 본인 GitHub 아이디로 바꾸세요)
git remote add origin https://github.com/<your-username>/obsidian-ai-assistant.git

# GitHub에 올리기
git push -u origin main
```

> **주의:** `<your-username>` 부분을 본인의 GitHub 사용자명으로 반드시 바꾸세요.  
> 예: `git remote add origin https://github.com/yeongyu/obsidian-ai-assistant.git`

GitHub 로그인을 요구하는 창이 뜨면 GitHub 계정 정보를 입력하세요.

---

### 5단계: 태그 push로 빌드 트리거

아래 명령으로 `v3.0.0` 태그를 만들어 GitHub에 push하면 빌드가 자동으로 시작됩니다.

```bash
# 태그 생성
git tag v3.0.0

# 태그를 GitHub에 push (빌드 시작!)
git push origin v3.0.0
```

---

### 6단계: 빌드 진행 확인

1. GitHub 저장소 페이지에서 **"Actions"** 탭 클릭
2. **"Release"** 워크플로우가 실행 중인 것을 확인할 수 있습니다.
3. 초록색 체크 표시가 뜨면 빌드 성공입니다.
4. 빌드는 보통 **10~20분** 소요됩니다. (macOS universal 빌드가 가장 오래 걸립니다)

빌드 중 오류가 발생하면 해당 항목을 클릭해 로그를 확인할 수 있습니다.

---

### 7단계: 완성된 설치 파일 다운로드

1. 저장소 페이지 → 오른쪽 **"Releases"** 클릭
2. `v3.0.0` 릴리즈 클릭
3. **"Assets"** 섹션 펼치기
4. 아래 파일들이 업로드되어 있습니다:

| 파일 | 대상 |
|-----|-----|
| `옵시디언.AI.비서_3.0.0_x64-setup.exe` | Windows (설치 프로그램) |
| `옵시디언.AI.비서_3.0.0_x64_en-US.msi` | Windows (MSI 패키지) |
| `옵시디언.AI.비서_3.0.0_universal.dmg` | macOS (M1/M2/M3/Intel 통합) |
| `옵시디언.AI.비서_3.0.0_amd64.deb` | Linux (Debian/Ubuntu) |
| `옵시디언.AI.비서_3.0.0_amd64.AppImage` | Linux (범용) |

> 릴리즈는 처음에 "Draft" 상태로 생성됩니다. GitHub 저장소에서 **"Edit"** → **"Publish release"**를 클릭해 공개로 전환할 수 있습니다.

---

### 이후 업데이트 빌드 방법

코드를 수정한 뒤 새 버전을 빌드하려면:

```bash
# 변경된 파일 스테이징
git add .

# 커밋
git commit -m "버전 3.0.1 업데이트"

# GitHub에 push
git push

# 새 태그 만들기 및 push (빌드 트리거)
git tag v3.0.1
git push origin v3.0.1
```

## B) 로컬 빌드 (개발자용)

직접 컴퓨터에서 빌드하고 싶은 개발자를 위한 방법입니다.

### 요구사항

| 도구 | 설치 방법 | 용도 |
|-----|---------|-----|
| **Rust** (stable) | https://rustup.rs/ | 핵심 빌드 도구 |
| **Node.js 18+** | https://nodejs.org | Tauri CLI 실행 |
| **npm** | Node.js에 포함 | 패키지 관리 |
| **macOS: Xcode CLT** | `xcode-select --install` | macOS 빌드 도구 |
| **Windows: WebView2** | Windows 11에 기본 포함 | Tauri 렌더링 엔진 |
| **Windows: MSVC** | Visual Studio Build Tools | Windows 컴파일러 |

### Rust 설치 (모든 OS 공통)

```bash
# rustup 설치 스크립트 실행
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 설치 후 환경변수 적용
source "$HOME/.cargo/env"

# 설치 확인
rustc --version
cargo --version
```

Windows에서는 https://rustup.rs 에서 `rustup-init.exe`를 다운로드해 실행합니다.

### macOS 추가 설정 (Apple Silicon + Intel 통합 빌드)

```bash
# Xcode Command Line Tools
xcode-select --install

# Universal 빌드용 타겟 추가
rustup target add aarch64-apple-darwin x86_64-apple-darwin
```

### 빌드 명령

```bash
# 프로젝트 폴더로 이동
cd "옵시디언 개인 ai"

# npm 의존성 설치 (Tauri CLI 포함)
npm install

# 개발 모드 실행 (핫 리로드 — 코드 수정 시 자동 반영)
npm run dev

# 프로덕션 빌드
npm run build

# macOS Universal 빌드 (Apple Silicon Mac에서 실행)
npm run build -- --target universal-apple-darwin
```

### 빌드 결과물 위치

빌드가 성공하면 아래 경로에 설치 파일이 생성됩니다:

```

├── msi/
│   └── 옵시디언.AI.비서_3.0.0_x64_en-US.msi      ← Windows MSI
├── nsis/
│   └── 옵시디언.AI.비서_3.0.0_x64-setup.exe       ← Windows 설치 프로그램
├── dmg/
│   └── 옵시디언.AI.비서_3.0.0_universal.dmg        ← macOS DMG
├── macos/
│   └── 옵시디언 AI 비서.app/                       ← macOS 앱 번들
├── deb/
│   └── obsidian-ai-assistant_3.0.0_amd64.deb       ← Linux Debian
└── appimage/
    └── obsidian-ai-assistant_3.0.0_amd64.AppImage  ← Linux AppImage
```

## C) 빌드 트러블슈팅

### "linker 'cc' not found" 오류 (Linux)

**원인:** 빌드에 필요한 시스템 라이브러리가 없습니다.

```bash
sudo apt-get update
sudo apt-get install -y \
  build-essential \
  libwebkit2gtk-4.1-dev \
  libappindicator3-dev \
  librsvg2-dev \
  patchelf
```

---

### "Tauri CLI not found" 또는 "tauri: command not found" 오류

**원인:** npm 패키지가 설치되지 않았습니다.

```bash
# 패키지 재설치
npm install

# 그래도 안 되면 node_modules 삭제 후 재설치
rm -rf node_modules
npm install
```

---

### macOS Universal 빌드 실패

**원인:** Intel 타겟이 추가되지 않았거나 macOS 버전이 낮습니다.

```bash
# 필요한 타겟 추가
rustup target add aarch64-apple-darwin x86_64-apple-darwin

# Universal 빌드 재시도
npm run build -- --target universal-apple-darwin
```

> Universal 빌드는 Apple Silicon Mac(M1/M2/M3)에서만 정상적으로 생성됩니다.  
> Intel Mac에서는 `x86_64-apple-darwin` 단일 타겟으로 빌드하세요.

---

### Windows "The code execution cannot proceed" 또는 런타임 오류

**원인:** Visual C++ 재배포 패키지가 없습니다.

1. https://aka.ms/vs/17/release/vc_redist.x64.exe 다운로드
2. 실행해서 설치
3. 재부팅 후 재시도

---

### Windows 코드 서명 경고 (사용자에게 경고 표시)

**원인:** 코드 서명 인증서 없이 빌드된 경우 사용자에게 "Windows에서 PC를 보호했습니다" 경고가 표시됩니다.

**해결 방법 (선택 사항):**
- 코드 서명 인증서 구입 (연간 약 $200~500)
- 또는 사용자에게 "추가 정보 → 실행"으로 우회하는 방법 안내

현재 개인 프로젝트 수준에서는 서명 없이 배포해도 사용에 문제없습니다.

---

### GitHub Actions 빌드가 실패해요

**확인 방법:**
1. GitHub 저장소 → **"Actions"** 탭
2. 실패한 워크플로우 클릭
3. 실패한 단계(빨간 X) 클릭
4. 오류 메시지 확인

**자주 나오는 오류:**

| 오류 메시지 | 해결 방법 |
|-----------|---------|
| `error[E0...]: cannot find ...` | Rust 코드 컴파일 오류. 코드 수정 필요 |
| `npm ERR!` | `package.json` 또는 `node_modules` 문제. `npm install` 재실행 |
| `tauri.conf.json` 관련 오류 | 설정 파일 형식 오류. JSON 유효성 검사 필요 |
| macOS 빌드만 실패 | `aarch64-apple-darwin` 타겟 설정 확인 |

---

### Rust 빌드 시간이 너무 오래 걸려요

처음 빌드 시 모든 의존성을 컴파일하므로 10~30분이 걸릴 수 있습니다.  
이후 빌드는 캐시가 적용되어 훨씬 빠릅니다.

GitHub Actions에는 `swatinem/rust-cache` 캐시가 설정되어 있어 두 번째 빌드부터는 5~10분으로 단축됩니다.

## 부록: 버전 번호 업데이트 방법

새 버전을 배포할 때 아래 파일들의 버전 번호를 동일하게 맞춰주세요.

| 파일 | 변경 위치 |
|-----|---------|
| `src-tauri/Cargo.toml` | `version = "3.0.0"` |
| `src-tauri/tauri.conf.json` | `"version": "3.0.0"` |
| `package.json` | `"version": "3.0.0"` |
| `src-tauri/src/lib.rs` | `fn app_version() -> &'static str { "3.0.0" }` |

그 다음 git tag를 새 버전 번호로 만들어 push합니다:

```bash
git add .
git commit -m "버전 3.1.0"
git tag v3.1.0
git push && git push origin v3.1.0
```
